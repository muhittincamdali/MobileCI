# MobileCI Roadmap

Last updated: 2026-04-01

## Goal

Make MobileCI the most trusted Swift-first CI/CD toolkit for Apple-centric mobile teams.

## Current Gaps

- the repo needs stronger proof against Fastlane and similar tools
- self-validation CI is weaker than the product promise
- enterprise trust signals need more public evidence
- advanced setup and edge-case documentation can be clearer

## Sprint 1: Proof And Reliability

- run `swift build` and `swift test` on every PR
- publish benchmark and comparison methodology
- document first-success setup for iOS, Flutter, and React Native
- publish one realistic end-to-end pipeline example

## Sprint 2: Enterprise Readiness

- improve test coverage on signing, caching, and deployment flows
- expand failure-mode documentation
- document security and supply-chain posture
- publish release and rollback guidance

## Sprint 3: Category Leadership

- create battle cards versus Fastlane and other CI tooling
- publish anonymized deployment proof
- expand sample templates and workflow packs
- turn benchmark results into a visible distribution asset
